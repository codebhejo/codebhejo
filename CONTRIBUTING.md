# Contributing to CodeBhejo

CodeBhejo is an open-source project focused on simple, secure, and efficient code sharing and collaboration for developers. Contributions from the community are highly appreciated and help make the project better for everyone.

---

## Code of Conduct

Please be respectful and considerate in all interactions. Harassment, abusive behavior, discrimination, or personal attacks are not tolerated.

By participating in this project, you agree to uphold a welcoming and professional environment.

---

## Ways to Contribute

You can contribute in many ways, including:

* Reporting bugs
* Suggesting new features or improvements
* Fixing issues
* Improving documentation
* Enhancing UI/UX
* Reviewing pull requests

---

## Reporting Bugs

If you find a bug:

1. Check existing issues to see if it has already been reported.
2. If not, open a new issue and include:

   * A clear description of the problem
   * Steps to reproduce
   * Expected vs actual behavior
   * Screenshots or logs if applicable

---

## Feature Requests

Feature ideas are welcome!

Please open an issue and describe:

* The problem you are trying to solve
* Your proposed solution
* Any alternatives you considered

---

## üîß Development Workflow

### 1. Fork the Repository

Fork the repo to your own GitHub account.

### 2. Clone Your Fork

```bash
git clone https://github.com/<your-username>/codebhejo.git
cd codebhejo
```

### 3. Create a Branch

```bash
git checkout -b feature/your-feature-name
```

Use descriptive branch names:

* `feature/add-auth`
* `fix/login-bug`
* `docs/update-readme`

---

## Testing

Before submitting your changes:

* Ensure the application builds successfully
* Run existing tests (if any)
* Manually test your changes

Pull requests that break existing functionality may be rejected.

---

## Commit Guidelines

* Write clear, concise commit messages
* Use present tense (e.g., "Add login validation")
* Keep commits focused on a single change

Example:

```text
feat: add public IP detection using WebRTC
```

---

## Submitting a Pull Request

1. Push your branch to your fork
2. Open a Pull Request against the `dev` branch
3. Clearly describe:

   * What your change does
   * Why it is needed
   * Any breaking changes
4. Link relevant issues if applicable

Your PR may be reviewed, and changes may be requested. Please be responsive and open to feedback.
